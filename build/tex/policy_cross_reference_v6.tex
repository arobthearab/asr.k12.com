\section{Policy Inventory Overview}

The Stride information security program comprises \textbf{21 Information Security Policies (ISPs)} aligned to NIST CSF 2.0 functions and \textbf{10 Integrated Information Security Policies (IISPs)} addressing cross-cutting operational domains. Together, these 31 policy documents (30 published + 1 reserved) establish the governance foundation for cybersecurity risk management.

\vspace{0.3cm}
\textbf{NIST CSF 2.0 Function Coverage:}
\begin{itemize}
    \item \textbf{GOVERN (GV):} ISP 1.0--1.2, ISP 2.1--2.6, IISP 4.0, 5.0, 8.0
    \item \textbf{IDENTIFY (ID):} ISP 3.1--3.3, IISP 7.0
    \item \textbf{PROTECT (PR):} ISP 4.1--4.5, IISP 2.0, 3.0, 9.0
    \item \textbf{DETECT (DE):} ISP 5.1--5.2
    \item \textbf{RESPOND (RS):} ISP 6.1--6.2, IISP 10.0
    \item \textbf{RECOVER (RC):} IISP 1.0
\end{itemize}

\textcolor{gapred}{\textbf{Gap:} IISP 6.0 is reserved and not yet published. No policy currently occupies this slot in the numbering scheme.}

\section{APPENDIX C --- SOX \S 404 ITGC Reverse Mapping}
\label{app:sox}

\subsection{Access Controls}

Restrict system access to authorized individuals; enforce least privilege and separation of duties

\textbf{Stride Policies:} \hyperref[pol:ISP23]{ISP 2.3}, \hyperref[pol:ISP26]{ISP 2.6}, \hyperref[pol:ISP41]{ISP 4.1}, \hyperref[pol:ISP43]{ISP 4.3}, \hyperref[pol:IISP20]{IISP 2.0}, \hyperref[pol:IISP80]{IISP 8.0}

\textbf{Representative Controls:}
\begin{itemize}[leftmargin=*]
  \item AC-1: Logical access provisioning/deprovisioning aligned with HR lifecycle events
  \item AC-2: Periodic user access reviews (quarterly for privileged, semi-annual for standard)
  \item AC-3: Privileged access management --- just-in-time elevation, session recording, break-glass procedures
  \item AC-4: Authentication controls --- MFA for all administrative and financially-significant system access
  \item AC-5: Service account governance --- inventory, ownership, credential rotation
\end{itemize}

\subsection{Change Management}

Ensure changes to IT systems are authorized, tested, and approved before production deployment

\textbf{Stride Policies:} \hyperref[pol:ISP24]{ISP 2.4}, \hyperref[pol:ISP44]{ISP 4.4}, \hyperref[pol:IISP70]{IISP 7.0}, \hyperref[pol:IISP90]{IISP 9.0}

\textbf{Representative Controls:}
\begin{itemize}[leftmargin=*]
  \item CM-1: Change request documentation including business justification and impact assessment
  \item CM-2: Segregation of development, test, and production environments
  \item CM-3: Code review / peer review before production deployment
  \item CM-4: Change approval by designated change authority (CAB or equivalent)
  \item CM-5: Emergency change procedures with post-implementation review
  \item CM-6: Release management and deployment automation controls
\end{itemize}

\subsection{Computer Operations}

Ensure reliable processing of financial data through job scheduling, backup, and monitoring

\textbf{Stride Policies:} \hyperref[pol:ISP31]{ISP 3.1}, \hyperref[pol:ISP43]{ISP 4.3}, \hyperref[pol:ISP45]{ISP 4.5}, \hyperref[pol:ISP51]{ISP 5.1}, \hyperref[pol:ISP61]{ISP 6.1}, \hyperref[pol:IISP10]{IISP 1.0}

\textbf{Representative Controls:}
\begin{itemize}[leftmargin=*]
  \item CO-1: Job scheduling controls --- automated scheduling with failure alerting
  \item CO-2: Backup and recovery --- automated backups with tested restoration procedures
  \item CO-3: Incident management --- defined escalation and resolution procedures
  \item CO-4: Batch processing integrity --- reconciliation and completeness checks
\end{itemize}

\subsection{Program Development}

Ensure systems are developed, tested, and implemented with appropriate controls

\textbf{Stride Policies:} \hyperref[pol:ISP44]{ISP 4.4}, \hyperref[pol:IISP50]{IISP 5.0}, \hyperref[pol:IISP90]{IISP 9.0}

\textbf{Representative Controls:}
\begin{itemize}[leftmargin=*]
  \item PD-1: System development lifecycle (SDLC) methodology with security requirements
  \item PD-2: Security testing --- SAST, DAST, SCA integrated into CI/CD pipeline
  \item PD-3: User acceptance testing (UAT) for financially-significant system changes
  \item PD-4: Data migration controls with reconciliation and validation
\end{itemize}
